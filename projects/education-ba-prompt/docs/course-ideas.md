# Идеи для курса: Промпт-инжиниринг для бизнес-аналитиков

Этот документ собирает сырой материал для будущего курса: термины, кейсы,
шаблоны, идеи модулей, упражнения и вопросы для обсуждения. Материал можно
перестраивать по мере накопления практики.

## 🔹 Термины и концепции

- Output stability: почему важно для ТЗ, как проверять повторяемость и где
  допустима вариативность ответа.
- Consistency: единая терминология в серии запросов, связь с глоссарием проекта
  и контроль противоречий.
- Controlled generation: шаблоны, JSON-mode, ограничения, примеры допустимых и
  недопустимых ответов.
- Structured output: схемы, чек-листы, таблицы, форматы для передачи результата
  в CRM, backlog или документ требований.
- Context boundary: что давать модели в запросе, что хранить в базе знаний, что
  не передавать из-за приватности или избыточности.
- Human review: какие решения БА обязан проверить вручную перед передачей
  заказчику или команде разработки.

## 🔹 Практические кейсы БА

- Валидация полей ТЗ через промпт.
- Генерация комментариев для DID-номеров.
- Синхронизация требований с CRM (amoCRM, Bitrix).
- Коммуникация с заказчиком: как формулировать запросы.
- Преобразование интервью или переписки в список требований с открытыми
  вопросами.
- Проверка требований на неоднозначность, неполноту и конфликт терминов.
- Подготовка release notes или change summary по списку задач.

## 🔹 Шаблоны промптов (рабочие)

- Шаблон для анализа ТЗ.
- Шаблон для генерации функциональных требований.
- Шаблон для валидации по чек-листу.
- Шаблон для уточняющих вопросов заказчику.
- Шаблон для сравнения текущей версии требований с новой вводной.
- Шаблон для структурированного вывода в таблицу или JSON.

## 🔹 Идеи упражнений

- Дать участнику фрагмент ТЗ и попросить найти неоднозначные требования через
  промпт с чек-листом.
- Сравнить свободный промпт и промпт со структурированным выводом на одном
  кейсе.
- Преобразовать переписку с заказчиком в требования, список рисков и вопросы.
- Настроить шаблон, который стабильно возвращает один формат для разных
  входных данных.
- Провести peer review промпта другого участника по критериям ясности,
  ограничений и проверяемости результата.

## 🔹 Форматы подачи

- Чат-бот с системным промптом + kb/.
- Воркшоп с живыми примерами.
- Самообучение: интерактивные задания с автопроверкой.
- Reference-first база шаблонов для повторного использования после обучения.
- Blended формат: короткая теория, практика в чате, разбор результата с
  преподавателем или ментором.
