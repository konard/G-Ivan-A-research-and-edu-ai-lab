
### 4.2 Критерии готовности промпта (из аудита)

1. Объясняет роль AI-ассистента
2. Определяет порядок диалога (workflow)
3. Содержит правила и ограничения
4. Определяет формат вывода
5. Явно запрашивает входные данные (CTA)
6. Размер: `_exp` ≤ 1000 символов, `_simple` ≤ 3000 символов

### 4.3 Наблюдаемые итерации отладки

| Итерация | Что менялось | Пример |
|----------|-------------|--------|
| 1. Структурирование | Добавление секций РОЛЬ / АЛГОРИТМ / ПРАВИЛА | Из свободного текста → structured prompt |
| 2. Разделение вариантов | `_exp` (RAG) vs `_simple` (standalone) | Один прототип → два production промпта |
| 3. Сжатие | Удаление примеров и логов из тела промпта | ~22K → ~1000/3000 символов |
| 4. Добавление stop-gates | Обязательные паузы перед финальной генерацией | «СТОП: подтвердите акторов» |
| 5. Контролируемый вывод | YAML-метаданные, таблицы, чек-листы | Свободный текст → структурированный output |

## 5. Выводы для классификации

### 5.1 Подтверждённые оси классификации

Анализ промптов в репозитории подтверждает релевантность всех 5 осей,
предложенных в issue #79:

1. **Тип промпта** (`simple` / `system` / `agent` / `multi-agent`) — различается
   по степени контроля над поведением и выводом.
2. **Доступ к знаниям** (`standalone` / `RAG-enabled`) — подтверждён двойным
   вариантом `_exp` / `_simple`.
3. **Режим диалога** (`single-shot` / `multi-turn` / `orchestrated`) — различается
   по наличию stop-gates и clarification loops.
4. **Сценарий применения** (`чат с человеком` / `агент` / `мульти-агент` / `API`) —
   в репозитории пока только «чат с человеком».
5. **Уровень зрелости проекта** (1–4) — все промпты на уровне 2; уровни 1, 3, 4
   не представлены.

### 5.2 Предложение дополнительной оси

| Ось | Значения | Обоснование |
|-----|----------|-------------|
| **Домен применимости** | `domain-specific` / `universal` | Все текущие промпты — Mango-specific; классификация должна покрывать и универсальные промпты |

### 5.3 Рекомендации для следующего шага

1. Использовать 5+1 осей для построения матрицы в
   `research/prompts-classification-standard-2026-05.md`.
2. Заполнить ≥8 ячеек матрицы, включая теоретические ячейки для уровней 1, 3, 4.
3. Создать 3 шаблона промптов (simple, system, agent) для тестирования практики
   «отладки формата».
4. Предложить план интеграции классификации к промптам в `projects/mango/prompts/`.
