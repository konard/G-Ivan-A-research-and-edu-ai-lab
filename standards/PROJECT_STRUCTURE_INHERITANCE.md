# Project Structure Inheritance

## Правило наследования структуры каталогов в проектах

### Базовый принцип

Проекты могут итеративно создавать подкаталоги, наследуя структуру репозитория,
но **без создания обязательных зависимостей** для других проектов.

### Разрешённые подкаталоги в `projects/{project}/`

| Подкаталог | Назначение | Наследует правила из | Область применения |
| --- | --- | --- | --- |
| `standards/` | Проект-специфичные стандарты (термины, шаблоны, правила) | `/standards/` (опционально) | Только этот проект |
| `kb/` | База знаний для промптов и аналитики | `standards/FILE_NAMING.md`, `standards/GLOSSARY.md` (опционально) | Только этот проект |
| `prompts/` | Системные промпты и оркестрация | `kb/`, проектные стандарты | Только этот проект |
| `docs/` | Документация, на которую ссылаются промпты | - | Только этот проект |
| `experiments/` | Тесты, A/B-сравнения, метрики | - | Только этот проект |
| `decisions/` | Зафиксированные архитектурные/процессные решения | - | Только этот проект |

### Правила связывания стандартов

1. **Проектный стандарт может ссылаться на репозиторий-широкий** (опционально):

   ```yaml
   # В frontmatter проектного стандарта
   related_standards:
     - "../../standards/GLOSSARY.md"  # Опциональная ссылка
   ```

2. **Репозиторий-широкий стандарт НЕ должен ссылаться на проектный** (защита
   от циклических зависимостей).
3. **Явная метка области применения** в frontmatter проектных стандартов:

   ```yaml
   scope: mango-only  # или: repo-wide, shared, experimental
   ```

### Пример структуры проекта

```text
projects/mango/
├── README.md                          # Навигация по проекту
├── standards/
│   ├── classification-glossary.md     # scope: mango-only
│   └── classification-hierarchy.md    # scope: mango-only
├── kb/
│   ├── product-matrix.md              # RAG-навигатор
│   └── glossary.md                    # Предметный глоссарий для промптов
├── prompts/
│   └── system.md                      # Системный промпт
├── docs/
│   └── analysis.md                    # Документация для ссылок
├── experiments/
│   └── prompt-ab-test/                # Тесты эффективности
└── decisions/
    └── architecture-2026-05.md        # Зафиксированные решения
```

### Проверка

- Новый проект создаёт подкаталог не из списка? -> обосновать в описании
  задачи.
- Проектный стандарт не имеет метки `scope`? -> добавить в ревью.
- Репозиторий-широкий стандарт ссылается на проектный? -> отклонить в ревью.
